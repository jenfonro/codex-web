# Frontend Structure

This frontend is intentionally static and framework-free for now, but its source layout follows the same responsibility split used by OpenList-Frontend:

- `app/`: Codex Web bootstrap behavior and app-level layout styles.
- `components/`: reusable UI helpers that are not tied to a single page.
- `pages/`: page or panel specific runtime code.
- `store/`: small state factories and persistence helpers.
- `assets/`: copied local assets and fixed runtime resources, grouped by source boundary.

The Codex panel lives under `pages/codex/`:

- `index.js`: panel bootstrap and event coordination.
- `config.js`: Shadow DOM mount and static resource list.
- `api.js`: controller API calls and response normalization.
- `fixtures.js`: explicit `?codexFixture=...` reference data for audits and visual regression only.
- `renderer.js`: current static DOM renderer for the copied Codex panel shape.
- `utils.js`: formatting, escaping, time, and activity helpers.
- `panel.css`: host-page adapter styles.
- `panel-shadow.css`: Shadow DOM adapter styles.

Shared Codex panel state lives in `store/codex.js`. Keep browser persistence and reusable state defaults there rather than in the page bootstrap.

The controller workspace file browser lives under `pages/workspace/`:

- `index.js`: calls `/api/workspace` for directory listing and file search on the active agent node.
- `workspace.css`: compact sidebar file-list styles.

The controller node management view lives under `pages/nodes/`:

- `index.js`: fetches `/api/nodes`, selects the active node, deletes offline nodes, and emits node-selection events for the Codex panel.
- `nodes.css`: compact sidebar list styles for controller-owned node management UI.

The controller Git view lives under `pages/git/`:

- `index.js`: calls `/api/git` for branch metadata, status counts, and changed file summaries on the active agent node.
- `git.css`: compact sidebar source-control styles.

The controller Runs view lives under `pages/runs/`:

- `index.js`: aggregates sessions from online agent nodes, subscribes to node session events, opens sessions back in the Codex panel, and cancels running turns through the controller.
- `runs.css`: compact sidebar run-list styles.

The outer workspace chrome is generated by `components/workspace/layout.js`.
Keep app-owned labels, ids, and data structures there; `index.html` should stay
as a mount point and asset list.

Copied Codex panel extension CSS under `assets/codex-panel/` must remain byte-for-byte aligned with the captured official assets unless the source-alignment audit is updated with an explicit adapter rule.
